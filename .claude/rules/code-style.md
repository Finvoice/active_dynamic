# Code Style (RuboCop)

- Configuration in `.rubocop.yml`, mirroring `finvoice_underwrite_api`'s rules minus the
  Rails-only plugins/cops (this is a plain gem). Target Ruby 3.4.
- Autocorrect with `make rubocop-auto-correct` (**safe-only `-a`**), not `-A`. Unsafe
  corrections can change behavior — e.g. `Style/SafeNavigation` rewrites `x && x.y` to
  `x&.y`, which differs when `x` is `false`. Use `make rubocop-auto-correct-unsafe` (`-A`)
  only when you review every change. CI runs `make rubocop` (`bundle exec rubocop`).
- Frozen string literal comments should be absent (enforced style: never).
- Use bracket syntax for word/symbol arrays, not `%w` / `%i`.
- Use `no_mixed_keys` hash syntax with shorthand when possible.
- Line-break enforcement is on for arrays, hashes, and method arguments.
- Documentation comments not required; max class length 200 (smart counting).

## .rubocop_todo.yml

Pre-existing offenses in the legacy gem code are grandfathered in `.rubocop_todo.yml`
(`inherit_from`), the same adoption pattern the underwrite repo uses. It was generated by
`make rubocop-update-todos` after a safe `-a` pass — so it only ignores offenses that
weren't safely autocorrectable. Some entries are **permanent** and must not be "fixed":
- `Naming/PredicatePrefix` on `has_dynamic_attributes` (the entry-point macro — public API)
- `Naming/ConstantName` on the `ActiveDynamic::DataType` constants (`String`, `Text`, … — public API)
- `Style/OptionalBooleanParameter` on `respond_to?` (must match Ruby's signature)
- `Style/SafeNavigation` on the dynamic setter in `has_dynamic_attributes.rb` (`x && x.y`
  keeps a literal `false` as-is; `x&.y` would coerce it to `"false"`)

The rest (Metrics, `Style/ClassVars`, `Style/MissingRespondToMissing`, the spec-only
`RSpec/DescribedClass` / `Style/GlobalStdStream`) are deferred cleanup — fix them in code
and drop the entry, don't expand the list. Do not silence cops in `.rubocop.yml` to dodge a
fix; fix the violation or use a scoped inline `# rubocop:disable` with a reason.
